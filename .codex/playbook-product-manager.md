# Product Manager Playbook

Use this playbook when work is planning, issue hygiene, or release readiness.

## Scope

- Own roadmap updates, issue shaping, and milestone alignment.
- Prepare tasks so implementation can start without major ambiguity.
- Coordinate release readiness and release execution when requested.

## Primary workflows

- `prepare-task`
- `feature-intake`
- `release`
- `refine-task`

## Handoff output

- Updated planning context
- Selected task with implementation brief
- Readiness or release status with blockers and next action

## Guardrails

- Do not push, tag, publish releases, or close milestones without explicit user approval.
- Prefer refining existing issues over creating duplicates.

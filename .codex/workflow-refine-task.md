# Workflow: Refine Task

## Use this when

- A task is valuable but too vague to implement safely.
- The next goal is removing ambiguity, not coding.

## Workflow

1. Identify readiness gaps.
- Goal ambiguity
- Scope edges
- Missing acceptance criteria
- Dependency uncertainty
2. Resolve gaps from existing repo context first.
3. Ask targeted questions only for true blockers.
4. Produce implementation-ready structure.
- Goal
- Scope
- Non-goals
- Acceptance criteria
- Affected systems/files
- Validation expectations
5. Update the existing planning artifact when possible.

## Guardrails

- Do not start implementation code.
- Avoid broad open-ended interviews when a targeted decision is enough.

# Workflow: Feature Intake

## Use this when

- The user proposes a new feature or improvement.
- The idea should be represented in roadmap and issue planning.

## Workflow

1. Confirm idea is specific enough to plan.
2. Read relevant roadmap section and related issues.
3. Place the idea.
- Current milestone when small and urgent
- Later milestone when valuable but not next
- Future vision when too large or premature
4. Prefer updating existing issue over creating duplicates.
5. Update `ROADMAP.md` when planning changes.
6. Create a local Conventional Commit for planning/doc updates when meaningful.

## Guardrails

- Do not write implementation code here.
- Do not publish releases or close milestones without explicit user approval.

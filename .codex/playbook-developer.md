# Developer Playbook

Use this playbook when work is primarily technical delivery.

## Scope

- Implement feature, bugfix, config, and docs changes in this repository.
- Follow `AGENTS.md` conventions and boundaries.
- Run the most relevant validation commands when practical.

## Primary workflows

- `implement-task`
- `explore-x4-api`
- `refine-task` (only when requirements are not implementation-ready)

## Handoff output

- What changed and why
- Validation executed and outcome
- Retest guidance for verification

## Guardrails

- Do not push, tag, or rewrite history without explicit user approval.
- Do not revert unrelated local changes.
